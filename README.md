# UAV Topological Barcode

Прототип алгоритма поиска визуальных ориентиров для навигации БПЛА на основе топологических признаков изображения.

Проект реализует поиск похожего изображения по ROI-фрагменту: из изображения строится компактный дескриптор, затем он сравнивается с базой эталонных ориентиров. В качестве основной версии используется `v0.5 texture100`: топологический barcode + edge/shape-признаки + texture/line-признаки.

## Цель проекта

Цель работы — проверить, можно ли использовать топологические признаки изображения как компактный дескриптор визуального ориентира.

Задачи проекта:

- подготовить набор эталонных ROI-изображений;
- сформировать искаженные query-изображения;
- построить топологический barcode по порогам яркости;
- дополнить barcode признаками границ, формы, текстуры и линий;
- сравнить query с эталонной базой;
- оценить точность Top-1 / Top-3 и время работы алгоритма;
- выбрать рабочую версию алгоритма для дальнейшей адаптации под микро-ПК.

## Идея алгоритма

Алгоритм работает не с исходным изображением целиком, а с ROI — областью интереса, где находится визуальный ориентир.

Общая схема:

1. Изображение переводится в grayscale.
2. По нескольким порогам яркости строятся бинарные представления.
3. На каждом бинарном уровне выделяются компоненты связности.
4. Для компонент отслеживаются интервалы существования по шкале порогов.
5. Эти интервалы формируют топологический barcode.
6. Barcode переводится в числовой вектор.
7. К barcode добавляются признаки границ, формы, текстуры и линий.
8. Вектор query сравнивается с векторами эталонов.
9. Чем меньше расстояние между векторами, тем выше объект в Top-K выдаче.

## Используемые признаки

### 1. Topological barcode

Barcode описывает, какие компоненты появляются и исчезают при изменении порога яркости.

Для каждой устойчивой компоненты сохраняются:

- `birth` — порог, на котором компонента появилась;
- `death` — порог, до которого компонента существовала;
- `persistence` — длительность существования;
- `max_area` — максимальная площадь компоненты.

### 2. Edge / shape features

В версии `v0.4` к barcode были добавлены признаки формы и границ. Это позволило лучше различать мосты, улицы, крыши и городские кварталы.

### 3. Texture / line features

В версии `v0.5` добавлены текстурные и линейные признаки:

- локальная яркость по блокам;
- локальная дисперсия по блокам;
- плотность границ;
- LBP-подобная текстурная гистограмма;
- признаки линий через HoughLinesP.

Именно `v0.5 texture100` выбрана как основная рабочая версия.
